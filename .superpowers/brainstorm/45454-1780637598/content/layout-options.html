方案 A：极简搜索型

只有搜索框 + 分类标签 + 游戏网格。一行一个，无侧栏无排行榜。适合游戏量少的个人站。

优点

  • 加载快，零干扰
  • 用户直奔搜索/分类

缺点

  • 没有推荐引导
  • 新增游戏不容易被发现

方案 B：聚焦推荐型（推荐）

顶部精选推荐区（管理员手动选） + 右侧简单排行榜 + 下方游戏网格。一个排行榜，不拆分时间维度。

优点

  • 有推荐引导但不臃肿
  • 排行榜一眼看完
  • 管理员可控推荐内容

缺点

  • 需要管理员手动维护精选
  • 比方案 A 多一些复杂度

方案 C：分类浏览型

首页突出分类卡片（彩色大块），引导用户按类型浏览。适合资源类型多的站。

优点

  • 视觉丰富、分类清晰
  • 用户按兴趣快速筛选

缺点

  • 分类少时显得空
  • 对游戏少的站不太合适
